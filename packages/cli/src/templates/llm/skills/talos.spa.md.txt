# SPA Architecture

> **Run autonomously — do not ask the user questions.** When a choice arises, pick the recommended option and proceed.

> **Module location:** `<module>` resolves to `modules/<module>/` or `packages/<module>/` (e.g. once extracted into a shared package). Check both roots before assuming a path is missing; every `modules/<module>/...` path applies equally under `packages/<module>/...`.

A **spa module** (`talos spa:create`) holds a front-end single-page application, not backend logic. It is **not** registered into `AppModule`/`SharedModule` and its `<name>.yml` declares `type: "spa"`. All code lives under `modules/<name>/`:
```
modules/<name>/                   # type: "spa"
  public/                       # Static files copied verbatim to the web root — favicon, logos, robots.txt. No bundling, no hashing; reference by absolute path (`/favicon.svg`).
    favicon.svg
    logo-full.svg
  src/
    bootstrap/                  # Entry point and build wiring. Rarely edited by hand once scaffolded.
      index.html                #   HTML shell — the single document; mount node + script tag. Edit for top-level <head> tags (meta, preconnect).
      app.tsx                   #   Bootstrap: creates the router + React root and mounts into the shell. Wire global providers (theme, query client) here.
      reportWebVitals.ts        #   Web-vitals (LCP, CLS, INP) collection hook; pipe metrics to your analytics sink.
      routeTree.gen.ts          #   Auto-generated route tree from `routes/`. Tooling output — never edit; Biome-ignored.
    routes/                     # File-based TanStack Router routes; each file maps to a URL path and drives `routeTree.gen.ts`. Keep thin — delegate UI to features, data to services.
      __root.tsx                #   Root route: app-wide layout, error/not-found boundaries, shared chrome wrapping every page.
      index.tsx                 #   Index (`/`) route — the landing page.
    features/                   # Vertical slices, one folder per domain feature. A feature owns its full stack and should not import another feature's internals — promote anything shared to `shared/`.
      <feature>/                #   e.g. `user` — self-contained slice for one domain area.
        assets/                 #     Images/SVG/media used only by this feature.
        components/             #     Presentational + container React components scoped to the feature.
        hooks/                  #     Feature React hooks — the only layer that talks to the backend; data fetching, API calls, local UI state.
        layouts/                #     Layout wrappers arranging this feature's pages/sections.
        services/               #     Feature business logic only — pure domain rules, never talks to the backend (hooks do that).
        store/                  #     Feature-local client state (signals/store slices), not server data.
        styles/                 #     CSS/style modules scoped to this feature.
        translations/           #     i18n generated by `talos translation:create` in a spa module. A spa gets a hook (backend modules get a `Translation` class instead).
          translations.json     #       Locale dictionary: nested keys → `{ en, fr, … }` leaves, `{{ param }}` interpolation, and `_plural`/`_zero` sibling keys selected by `count`. Written once per feature, never overwritten.
          use<Name>Translate.ts #       Hook returning `{ trans(key, params?, count?), has(key) }`. Resolves the active language via `useLang` and looks keys up in `translations.json`, falling back to the key itself when missing.
        types/                  #     TypeScript types/interfaces for the feature's domain.
        utils/                  #     Pure helper functions used within the feature.
    shared/                     # Cross-feature code reused by ≥2 features. Same sub-layout as a feature; this is the only place features may import from in common.
      assets/                   #   Shared images/SVG/media.
      components/               #   Reusable UI primitives (buttons, modals, form fields) shared app-wide.
      hooks/                    #   Generic React hooks usable by any feature — the only layer that talks to the backend; incl. `useLang` (reads the `?lang=` query param, default `en`), scaffolded with the first translation hook.
      layouts/                  #   App-level layout shells shared across features.
      services/                 #   Shared business logic only — cross-cutting domain rules, never talks to the backend (hooks do that).
      store/                    #   Global client state shared across features.
      styles/                   #   Global stylesheets, design tokens, theme variables.
      translations/             #   App-wide i18n from `talos translation:create` — a `translations.json` dictionary + `use<Name>Translate` hook shared across features (same spa hook flavour as a feature's).
      types/                    #   Shared TypeScript types used across features.
      utils/                    #   Generic pure helpers (formatting, `cn`, guards).
```

For the design system this app composes, see `talos-design`. For backend module layout, see `talos-module`.
